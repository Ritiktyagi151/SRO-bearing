# Security and Malware Audit Report

Date: 2026-06-08

## Scope

Reviewed the full project tree, including hidden files, source files, public assets, build output, configuration files, lockfiles, and installed dependencies. Generated/vendor folders (`node_modules`, `.next`, `out`) were scanned with dependency tooling and malware-pattern searches rather than manually edited.

## Executive Summary

- Malware found: none confirmed.
- Infected files: none confirmed.
- Suspicious code fixed: one quote-form endpoint in `src/components/Navbar.jsx` posted to a personal Gmail address and was changed to the company endpoint already used in `src/components/Navbar4.jsx`.
- Dependency vulnerabilities before fix: 14 total from `npm audit` (1 critical, 6 high, 6 moderate, 1 low).
- Dependency vulnerabilities after fix: 0 total from `npm audit`.
- Visual/UI impact: none intended; no layout, content, image, color, spacing, typography, animation, SEO, or route structure was changed.

## Infected Files

No confirmed malware-infected files were found.

Files with reviewed security-relevant code:

- `src/components/Navbar.jsx`: unauthorized/suspicious form endpoint fixed.
- `src/components/Navbar4.jsx`: legitimate company FormSubmit endpoint reviewed.
- `src/pages/contact.jsx`: Google Maps iframe reviewed; no hidden redirect found.
- `src/components/homesections/Homeabout.jsx`: local `/products` navigation reviewed.
- `src/pages/blogs/[slug].jsx`: `dangerouslySetInnerHTML` reviewed; content is hardcoded local blog HTML, not user input.

## Vulnerable Dependencies Report

Initial `npm audit` findings included:

- `next@15.4.1`: critical/high/moderate advisories including React Flight RCE, DoS, SSRF, image optimizer, cache poisoning, and middleware/proxy issues.
- `react-router-dom@7.6.3` / `react-router`: high/moderate advisories; package was unused in source.
- Transitive advisories: `postcss`, `@eslint/plugin-kit`, `ajv`, `brace-expansion`, `flatted`, `glob`, `js-yaml`, `minimatch`, `picomatch`, `tar`, `yaml`.

After remediation:

- `npm audit --json`: 0 vulnerabilities.
- `react-router-dom` was removed because it was not imported anywhere.
- `next` lockfile resolved version updated to `15.5.19`.
- `eslint-config-next` updated to `15.5.19`.
- `postcss` resolved to `8.5.15`.
- Added targeted npm `overrides` for patched transitive versions:
  - `brace-expansion@1.1.15`
  - `flatted@3.4.2`
  - `glob@10.5.0`
  - `minimatch@3.1.5`
  - `picomatch@4.0.4`
  - `postcss` pinned to the direct dependency
  - `yaml@2.8.3`

Outdated packages intentionally not major-upgraded:

- `next` latest major is `16.2.7`; kept on patched Next 15 line to avoid framework migration risk.
- `eslint` latest major is `10.4.1`; kept on ESLint 9 line.
- `tailwindcss` latest major is `4.3.0`; kept direct Tailwind 3 line to preserve styling behavior.
- `lucide-react` and `react-intersection-observer` have newer majors; not changed because no vulnerability required it.

## Recommendations

- Replace third-party placeholder/media URLs with owned or licensed local assets where practical to reduce supply-chain/media availability risk.
- Move FormSubmit email destinations into environment configuration when deployment supports it.
- Consider adding a Content Security Policy with explicit `img-src`, `media-src`, `frame-src`, and `connect-src` allowlists.
- Remove the unused default `/api/hello` endpoint if it is not needed in production.
- Migrate `next lint` to the supported ESLint CLI before upgrading to Next 16.
- Periodically run `npm audit`, `npm outdated`, and a lockfile review in CI.
